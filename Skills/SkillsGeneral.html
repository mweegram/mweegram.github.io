Software Source Control (Git, etc)

As part of my self-guided study of software design and development I have gathered an understanding for the need and benefits of using source control software like Git. I have begun using source control in my personal projects as well as within university programming projects.

Git/Github allows me to keep copies of the scripts I create and add to them without breaking or losing the original copies. When making changes I make sure to leave detailed commit messages and create branches to ensure that changes do not have any adverse effects on the main code of the file.

Git allows for group working on one code base, making sure changes are applied thoughfully. Code reviews and GPG key signed commits are a neccessity when working on group projects to ensure quality and integity of commits respectively.
